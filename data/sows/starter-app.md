# Statement of Work — Starter Mobile App

**Project:** Starter — Habit Tracking Mobile App
**Client:** Starter Inc.
**Start Date:** July 1, 2025
**End Date:** October 31, 2025
**Budget:** $80,000

---

## 1. Project Overview

Design and develop a cross-platform mobile application (iOS and Android) for daily habit tracking with push notification reminders and weekly progress reports.

## 2. Deliverables

### 2.1 Design Phase
- User research: 5 user interviews + persona creation
- UX wireframes for 12 core screens
- High-fidelity UI designs (Figma) with design system
- Prototype for user testing
- 2 rounds of design revisions

### 2.2 Development Phase
- Cross-platform app using React Native
- User authentication (email + Google Sign-In)
- Habit creation, editing, and deletion
- Daily check-in with streak tracking
- Push notification reminders (configurable per habit)
- Weekly progress summary (in-app)
- Settings screen (notification preferences, account management)
- Onboarding flow (3 screens)

### 2.3 Backend
- Firebase Authentication
- Firestore database for user data and habits
- Cloud Functions for push notification scheduling
- Basic REST API for admin dashboard (read-only)

### 2.4 Quality Assurance
- Unit tests for core business logic
- Integration testing on iOS and Android devices
- Beta testing via TestFlight (iOS) and Firebase App Distribution (Android)

### 2.5 Launch
- App Store submission (iOS)
- Google Play Store submission (Android)
- Post-launch: 4 weeks of bug fix support

## 3. Exclusions

The following items are explicitly NOT included:

- **Social features** — No friend lists, shared habits, leaderboards, or community features
- **Gamification** — No points, badges, rewards, or achievements system
- **In-app purchases or subscriptions** — No payment processing, premium tiers, or monetization features
- **Apple Watch / Wear OS companion app** — No wearable integrations
- **Offline-first architecture** — App requires internet connection
- **Data export** — No CSV/PDF export of habit data
- **Admin dashboard UI** — Only backend API; no front-end admin panel
- **Analytics dashboard** — No user analytics or business intelligence features
- **Multi-language support** — English only
- **Web version** — Mobile only, no web app
