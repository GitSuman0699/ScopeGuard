Statement of Work - BrightPath E-Learning Platform

Project: BrightPath - Corporate Training Portal
Client: BrightPath Learning Inc.
Start Date: July 15, 2025
End Date: November 30, 2025
Budget: $120,000


1. Project Overview

Design and develop a web-based corporate training platform where companies can onboard employees through structured courses, quizzes, and progress tracking.


2. Deliverables

2.1 Design Phase
- User research and persona development (3 personas)
- UX wireframes for 15 core screens
- High-fidelity UI design with design system (Figma)
- Admin dashboard design
- 2 revision cycles

2.2 Frontend Development
- Responsive web application using React
- Course catalog with search and filters
- Video player for lesson content (embedded YouTube/Vimeo only)
- Quiz engine (multiple choice, true/false)
- Progress tracking dashboard for learners
- Certificate generation (PDF) upon course completion

2.3 Admin Panel
- Course creation and management (CRUD)
- User management (invite, deactivate, assign roles)
- Analytics dashboard (completion rates, quiz scores)
- Bulk user import via CSV

2.4 Backend
- Node.js REST API
- PostgreSQL database
- JWT-based authentication (email + password)
- Role-based access control (Admin, Instructor, Learner)
- File upload for course materials (max 50MB per file)

2.5 Quality Assurance
- Unit and integration testing
- Cross-browser testing
- Load testing (up to 500 concurrent users)

2.6 Deployment
- Deployment to AWS (EC2 + RDS)
- CI/CD pipeline setup
- SSL certificate configuration
- 4 weeks post-launch bug fix support


3. Exclusions

The following items are explicitly NOT included:

- Mobile applications: No native iOS or Android apps
- Live video conferencing: No Zoom, Teams, or WebRTC integration
- AI-powered features: No AI tutoring, chatbot, or content recommendations
- SCORM or xAPI compliance: No learning standard integrations
- Payment processing: No subscriptions, billing, or payment gateway
- Content creation: Client provides all course content, videos, and materials
- Single Sign-On (SSO): No SAML, OAuth with external identity providers
- Multi-tenancy: Single organization only, no white-labeling
- Offline access: Requires internet connection
- Localization: English only, no multi-language support
- Gamification: No badges, points, leaderboards, or rewards
- Discussion forums: No community features, comments, or messaging between users
